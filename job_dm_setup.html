Generate DM Setup

Select the room types and construction types to write into Design Master, then download a .vbs setup script. Run it in the job folder (next to dm_hvac*.dm) with the drawing closed in AutoCAD. The script backs up the .dm first and skips any type already present, so it is safe to re-run.

{# ── Project settings (overwrite tblGlobal in the .dm) ──────────────── #}

Project settings overwrite these in the .dm

Prefilled from the work order where available. Blank fields are left unchanged in the Design Master file. The project name renames the project in DM.

The script targets the dm_hvac*.dm in the job folder and does not rename the file.

{# ── Site & solar data (NOT written to the .dm — Design Master has no field for #} {# these; ASHRAE Ch.14 values kept for the app's own records / load-calc payload) #}

Site & solar data saved to this job only — Design Master has no field for these

Design Master's own schema (and HTML export) only carries latitude, elevation, heating/cooling design dry-bulb, mean coincident wet-bulb, and mean daily range — the fields above. It has no longitude, standard meridian, dehumidification humidity ratio, or clear-sky τb/τd fields, so these are kept here for the app's own solar/psychrometric calculations only.
